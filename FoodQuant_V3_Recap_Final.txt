2.2 - Menu hebdomadaire
────────────────────────

  Système complet de planification sur 7 jours avec 4 choix par repas.
  L'utilisateur sélectionne ses plats, valide sa semaine, et les données
  remontent dans le Dashboard pour les stats et badges.

  Fichiers créés :
    - menu_hebdo.php   → Backend, 4 actions : générer, sélectionner, valider, courses
    - MaSemaine.jsx    → Frontend calendrier hebdomadaire

  Table MySQL : menu_hebdomadaire
    - utilisateur_id, semaine, jour, type_repas, choix_json, selection_index

  Fonctionnalités :
    - Navigation semaine précédente/suivante
    - Barre de progression X/28
    - Badge ✓ par jour complet
    - Bouton « Valider ma semaine »
    - Injection dans historique_repas (Dashboard/badges)
    - Liste de courses hebdo consolidée (arrondis réalistes)
    - Cartes cliquables → lien Marmiton quand sélectionné
    - Écran de chargement animé avec messages rotatifs


2.3 - Liste de courses intelligente
─────────────────────────────────────

  Refonte complète de la modale liste de courses (MaJournee.jsx + MaSemaine.jsx) :

    - Parsing des quantités (fractions 1/2, décimaux, unités collées 100g)
    - Cumul automatique (poulet lundi + poulet jeudi = total)
    - Arrondis réalistes :
        → Œufs, oignons → entiers (pas de 0.7 œuf)
        → Grammes : ≤50g → par 5g, ≤200g → par 10g, >200g → par 50g
        → Centilitres → par 5cl
        → Litres → par quart (0.25L, 0.5L, 0.75L, 1L)
        → Cuillères → par demi
    - Tri intelligent (ingrédients multi-repas en premier)
    - Fractions lisibles (¼, ½, ¾)
    - Appliqué sur les DEUX listes : Ma Journée + Ma Semaine


2.4 - Bot recettes françaises
───────────────────────────────

  32 recettes françaises intégrées en BDD (8 par type de repas) :
    - 8 petit-déjeuners (porridge, omelette, tartines avocat, pancakes...)
    - 8 déjeuners (poulet grillé, saumon quinoa, curry lentilles, risotto...)
    - 8 collations (energy balls, pomme-cacahuète, smoothie, houmous...)
    - 8 dîners (cabillaud brocoli, soupe lentilles, quiche, tofu sauté...)

  Avec ingrédients, instructions et macros complètes (cal/prot/lip/glu/fib).

  Les recettes locales sont prioritaires dans le moteur de menus :
    1) Recettes locales (BDD) → françaises, gratuites, cohérentes
    2) Cache Spoonacular
    3) API Spoonacular
    4) Recettes de secours (fallback)

  Les recettes locales affichent un placeholder emoji (dégradé ambré + emoji
  du type de repas) au lieu d'une fausse image Spoonacular.

  Fichier : bot_recettes_fr.php
    - ?action=sync       → Insère les recettes en BDD
    - ?action=sync&reset=1 → Réinitialise et réinsère
    - ?action=status     → Comptage par type
    - ?action=search&q=  → Recherche locale
    - ?action=random     → Recettes aléatoires (pour moteur menus)


2.5 - Traduction gratuite (MyMemory)
──────────────────────────────────────

  Remplacement de l'API Claude Haiku (payante) par MyMemory API :
    - Gratuit, 5000 mots/jour, aucune clé nécessaire
    - Le cache existant des traductions Claude est conservé
    - Vérification de qualité (texte identique, longueur, score)

  Système de fallback en 3 niveaux :
    1) MyMemory API → traduction automatique
    2) Cache local  → si déjà traduit auparavant
    3) Lien Google Translate → cliquable, pré-rempli

  Variable ANTHROPIC_API_KEY supprimable des variables Railway.


2.6 - Redirection Marmiton
────────────────────────────

  Le bouton « 📖 Recette » redirige vers une recherche Marmiton avec le
  titre français du plat :
    https://www.marmiton.org/recettes/recherche.aspx?aqt=Poulet+grillé+aux+herbes

  Appliqué sur les DEUX pages :
    - Ma Journée (bouton 📖 Recette sur chaque carte)
    - Ma Semaine (lien "📖 Voir recette" sous les cartes sélectionnées)

4. BASE DE DONNÉES MySQL (4 TABLES)
================================================================================

4.1 - Table « utilisateurs » (existante)
──────────────────────────────────────────

  idutilisateurs    INT AUTO_INCREMENT PK     Identifiant unique
  prenom            VARCHAR(100)              Prénom de l'utilisateur
  email             VARCHAR(255) UNIQUE       Adresse email
  mot_de_passe      VARCHAR(255)              Hashé avec bcrypt
  age               INT                       Âge
  weight            FLOAT                     Poids en kg
  height            FLOAT                     Taille en cm
  gender            VARCHAR(10)               homme / femme
  goal              VARCHAR(20)               perte / maintien / muscle
  diet              VARCHAR(30)               classique / végétarien / végan / sans gluten
  besoin_calorique  INT                       Calcul Mifflin-St Jeor
  date_inscription  DATETIME                  Date de création du compte


4.2 - Table « historique_repas » (existante)
─────────────────────────────────────────────

  id                INT AUTO_INCREMENT PK     Identifiant unique
  utilisateur_id    INT FK                    Référence utilisateurs
  date_repas        DATE                      Date du jour
  recettes_json     MEDIUMTEXT                4 recettes du jour en JSON (~1Ko)
  total_calories    INT                       Somme calories du jour
  total_proteines   INT                       Somme protéines du jour
  created_at        DATETIME                  Date de création
  UNIQUE KEY        (utilisateur_id, date_repas)


4.3 - Table « menu_hebdomadaire » (V3)
────────────────────────────────────────

  id                INT AUTO_INCREMENT PK     Identifiant unique
  utilisateur_id    INT FK                    Référence utilisateurs
  semaine           VARCHAR(10)               Format ISO : 2026-W12
  jour              TINYINT                   1 (lundi) à 7 (dimanche)
  type_repas        VARCHAR(20)               breakfast / lunch / snack / dinner
  choix_json        MEDIUMTEXT                JSON : tableau de 4 recettes proposées
  selection_index   TINYINT NULL              Index 0-3 du choix de l'utilisateur
  created_at        DATETIME                  Date de création
  updated_at        DATETIME                  Dernière modification
  UNIQUE KEY        (utilisateur_id, semaine, jour, type_repas)


4.4 - Table « recettes_locales » (V3)
───────────────────────────────────────

  id                INT AUTO_INCREMENT PK     Identifiant unique
  titre             VARCHAR(500)              Nom de la recette
  titre_normalise   VARCHAR(500)              Lowercase, sans accents (dédoublonnage)
  image_url         VARCHAR(1000)             URL image (ou vide → placeholder emoji)
  source_url        VARCHAR(1000)             URL source (optionnel)
  ingredients_json  MEDIUMTEXT                JSON des ingrédients
  instructions      TEXT                      Étapes de préparation
  calories          INT                       Kilocalories
  proteines         FLOAT                     Protéines en grammes
  lipides           FLOAT                     Lipides en grammes
  glucides          FLOAT                     Glucides en grammes
  fibres            FLOAT                     Fibres en grammes
  type_repas        VARCHAR(20)               breakfast / lunch / snack / dinner
  langue            VARCHAR(5) DEFAULT 'fr'   Langue de la recette
  created_at        DATETIME                  Date d'insertion
  UNIQUE KEY        (titre_normalise, type_repas)

7. DIFFICULTÉS RENCONTRÉES ET SOLUTIONS (15 MARS)
================================================================================

  Problème : Footer en doublon (affiché 2 fois)
    → Solution : Suppression du Footer dans Accueil.jsx, géré uniquement par App.jsx

  Problème : NavBar en bas masquait le footer sur mobile
    → Solution : NavBar déplacée en haut (fixed top-0), padding-top compensatoire

  Problème : Collation inadaptée (salade en collation)
    → Solution : Recettes locales françaises prioritaires avec vrais snacks

  Problème : Images incohérentes sur les recettes locales
    → Solution : Placeholder emoji (dégradé ambré + emoji du type de repas)

  Problème : Quantités irréalistes dans la liste de courses (0.17 avocat)
    → Solution : Parsing intelligent + arrondis réalistes (1 avocat)

  Problème : Connexion bloquée sur "Chargement..."
    → Cause : Incident réseau Railway (temporaire, pas lié au code)

  Problème : verifyJWT undefined dans menu_hebdo.php
    → Solution : Utilisation de verifierJWT (nom réel de la fonction dans jwt.php)

  Problème : Warnings Tailwind v4 (bg-gradient, flex-grow, min-h-[44px])
    → Solution : bg-linear-to-t, grow, min-h-11
